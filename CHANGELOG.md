## [1.0.0] - 2026-03-19

### Added
- Створено окремий робочий інтерфейс спеціаліста технічної підтримки в файлі `support_dashboard.html`.
- Реалізовано єдину стилістику з наявними `main_dashboard.html` та `kln_dashboard.html`.
- Додано верхню службову зону з назвою модуля, кількістю активних чатів і станом навантаження.
- Додано ліву панель зі списком чатів спеціаліста.
- Додано центральну область листування з історією повідомлень.
- Додано нижню область введення відповіді.
- Додано демо-стани інтерфейсу:
  - немає активних чатів
  - список чатів без відкритого діалогу
  - активний відкритий чат
  - велика кількість чатів
- Додано компактний вигляд карток чату:
  - ініціали користувача в аватарі
  - короткі бейджі `NEW`, `WAIT`, `LIVE`
  - індикатор непрочитаних повідомлень
  - час останньої активності
  - SLA/таймер очікування відповіді
- Додано локальний backend на `Express` у файлі `server.js`.
- Додано інтеграцію з Telegram через `Telegraf`.
- Додано admin mini panel у самому інтерфейсі для:
  - збереження `Bot Token`
  - введення `Public Webhook URL`
  - виставлення webhook
  - ручного оновлення списку чатів
- Додано локальне збереження налаштувань Telegram у `storage/telegram-settings.json`.
- Додано локальне збереження чатів і повідомлень у `storage/telegram-chats.json`.
- Додано endpoint для проксі-віддачі Telegram-файлів без витоку токена в браузер.
- Додано підтримку локального запуску через `localhost:3000/support`.
- Додано інструкцію запуску в `LOCAL_TELEGRAM_SETUP.md`.
- Додано `package.json` для запуску backend і встановлення залежностей.
- Додано `.gitignore` для локального сховища та `node_modules`.

### Changed
- Список чатів перероблено в компактний формат без зайвого прев’ю-тексту.
- Активний чат візуально підсвічено м’яким зеленим фоном.
- Перероблено блок `Стан зміни`, щоб картки не вилазили за межі сайдбару.
- Вирівняно висоту блоку `Чати спеціаліста` і центральної панелі листування.
- Для обох колонок додано внутрішній скрол при великому обсязі даних.
- Область листування зафіксовано по висоті, щоб нові повідомлення не розтягували layout.
- Додано автоматичне оновлення чатів без ручного рефрешу.
- Зменшено інтервал оновлення до швидшого режиму для майже live-відчуття.
- Додано щосекундне оновлення таймерів у браузері.
- Логіку статусу очікування відповіді змінено:
  - таймер стартує від останнього вхідного повідомлення клієнта після останньої відповіді спеціаліста
  - чат не вважається закритим лише через факт старої відповіді в історії
- Оновлено обробку `Enter` / `Shift+Enter` у полі відповіді:
  - `Enter` відправляє повідомлення
  - `Shift+Enter` додає новий рядок
- Оновлено рендер повідомлень спеціаліста, щоб зберігалися переноси рядків.
- Додано автоскрол до останнього повідомлення при надходженні нового повідомлення або відкритті чату.

### Media Support
- Додано обробку і відображення:
  - фото
  - GIF / animation
  - video/document media
  - звичайних стікерів
  - video stickers
- Для animated sticker `.tgs` додано базовий fallback-показ як службовий блок, без повного Lottie-рендеру.
- Оптимізовано показ GIF/коротких анімацій:
  - прибрано зайві контролли там, де вони не потрібні
  - додано стабільніший контейнер, щоб повідомлення не стрибало по висоті
  - зменшено кількість зайвих перерендерів, що перезапускали медіа

### Fixed
- Виправлено ситуацію, коли чат міг помилково показуватись як такий, де відповідь уже є, хоча останнє повідомлення було від клієнта.
- Виправлено логіку запуску таймера тільки після перезапуску сервера: тепер таймер і статус переходять у `WAIT` одразу після нового вхідного повідомлення.
- Виправлено відображення переносів рядків у повідомленнях спеціаліста.
- Виправлено автопрокрутку активного чату до останнього повідомлення.
- Виправлено переповнення карток у блоці `Стан зміни`.
- Виправлено розтягування інтерфейсу при великій кількості чатів або повідомлень.

### Known Limitations
- Для реального webhook потрібен публічний URL, наприклад через `ngrok`; `localhost` напряму недоступний для Telegram.
- Animated stickers формату `.tgs` ще не рендеряться нативно як повноцінна анімація.
- Зараз використовується polling + локальний live-таймер, а не WebSocket/SSE.
- Ролі, авторизація, розподіл чатів, база даних і production-ready багатокористувацька логіка ще не реалізовані.

## [1.1.0] - 2026-03-22

### Added
- Додано файлове сховище користувачів у `storage/users.json` без використання бази даних.
- Додано ролі доступу:
  - `administrator`
  - `specialist`
- Додано сесійну авторизацію через cookie.
- Додано API для:
  - входу в систему
  - перевірки активної сесії
  - створення користувачів
  - зміни ролі та пароля користувача
- Додано login overlay у `support_dashboard.html`.
- Додано окрему admin-секцію в інтерфейсі для:
  - керування користувачами
  - керування Telegram-настройками
- Додано стартові тестові облікові записи:
  - `admin / admin123`
  - `specialist / specialist123`

### Changed
- Адміністративні дії обмежено роллю `administrator`.
- Спеціаліст більше не повинен бачити адміністративну панель.
- Робочий дашборд переводиться в авторизований режим через перевірку сесії перед завантаженням чатів.

### Notes
- Сховище користувачів лишається файловим, що зручно для локального середовища та швидких ітерацій `v1.1.x`.
- Повноцінна production-аутентифікація, аудит дій і БД-персистентність лишаються на наступні версії.

## [1.1.1] - 2026-03-22

### Added
- Додано персоналізацію чатів по спеціалістах через закріплення чату за конкретним користувачем.
- Додано файлове збереження призначення чату у JSON-сховищі без БД.
- Додано admin API для призначення чату на спеціаліста.
- Додано завантаження списку спеціалістів для призначення чату з інтерфейсу.

### Changed
- Спеціаліст тепер бачить лише чати, закріплені саме за ним.
- Адміністратор бачить загальну кількість усіх чатів.
- У картці чату для адміністратора відображається поточний закріплений спеціаліст.
- В активному чаті додано елемент призначення чату на конкретного спеціаліста.

### Notes
- Нові чати за замовчуванням створюються як не призначені.
- Непризначені чати бачить адміністратор, а спеціалісти не бачать їх, доки чат не буде закріплено на них.
