Адміністративна секція доступна лише адміністратору. Тут можна керувати Telegram-інтеграцією та користувачами системи.
Online Specialists
Окремий список спеціалістів, які зараз у мережі, з показом кількості відкритих чатів у роботі.
Distribution Logic
Нові звернення автоматично призначаються на спеціаліста з найменшою кількістю відкритих чатів. Якщо цей клієнт уже мав діалог зі спеціалістом, він буде в пріоритеті за умови мінімального навантаження.
Auto Messages
Підтримуються плейсхолдери: `{specialistName}` для привітання та `{timeOfDay}` для завершення чату.
Telegram Admin
Token: не задано Webhook: не активовано Bot: demo
Для реального webhook Telegram потрібен публічний URL, бо `localhost` недоступний ззовні. Наприклад, підніми `ngrok http 3000`, встав URL сюди й натисни "Виставити webhook".
Після запуску локального сервера сторінка за адресою `http://localhost:3000/support` автоматично підтягне збережені налаштування і реальні чати з Telegram. Якщо сервер не запущений, інтерфейс залишиться на демо-даних.
Майбутнє розширення

Структура вже готова під Telegram-інтеграцію, автоматичне надходження нових звернень, робочі статуси чатів, службові позначки, історію звернень та передачу діалогу іншому спеціалісту.

UX-фокус

Інтерфейс не перевантажено другорядними блоками: у пріоритеті список чатів, кількість активних діалогів, поточний чат та поле відповіді. Це спрощує щоденну операційну роботу.

Адаптивність

Макет коректно перебудовується для ноутбуків і десктопів: при зменшенні ширини колонки стають вертикальними, але логіка одного робочого простору зберігається.
